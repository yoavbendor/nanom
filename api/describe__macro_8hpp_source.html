2// nanom — NANOM_DESCRIBE: the C++23 struct-registration macro (preprocessor provider).
3//
4// This is one of nanom's two `describe<T>` providers. The seam is nanom.hpp's
5// `describe<T>::fields()` -> std::tuple<detail::fld<Name, MemPtr>...>; this file synthesizes that
6// tuple with the preprocessor, nanom26.hpp synthesizes the identical tuple with C++26 P2996
7// reflection. The core library consumes only the seam and cannot tell the providers apart.
8//
9// Under C++26 reflection (NANOM_HAS_REFLECTION, see nanom.hpp), NANOM_DESCRIBE degrades to a
10// static_assert that reflection covers the type — every legacy registration line becomes a proof,
11// not a no-op. Define NANOM_DESCRIBE_FORCE_MACRO to restore the explicit specialization even under
12// reflection (an explicit specialization always beats nanom26's constrained partial
13// specialization, so this is also the per-type override for fields that reflection must not pick
14// up, e.g. registering a subset of members).
15//
16// This header is self-contained by design: it defines only macros, so it can be included before
17// nanom.hpp (the expansions reference ::nanom:: symbols at the USE site, where nanom.hpp must have
18// been included).
19
